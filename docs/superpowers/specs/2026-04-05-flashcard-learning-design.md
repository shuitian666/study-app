# 闪记学习模式设计文档

## 概述

将闪记功能与学习流程深度融合，打造类似Duolingo/扇贝单词的沉浸式学习体验。先闪后题，遗忘曲线驱动复习时机。

---

## 一、核心流程

### 学习顺序
1. 用户点击"开始学习"进入全屏闪记模式
2. 卡片正面显示知识点名称
3. 用户思考后点击翻转，看到背面（解析 + 例题 + 记忆技巧）
4. 底部显示3个按钮：不会 / 有点印象 / 会
5. 用户表态后，才能翻页（上/下一题）
6. 根据遗忘曲线，在适当时机出现已学过知识点的关联题目
7. 完成所有学习后，显示学习报告

### 遗忘曲线驱动
- 15个闪记示例：
  - 学到第5个闪记时 → 第1个知识点的关联题目出现
  - 学到第10个闪记时 → 可能出现第2个知识点的题目
  - 如果某个题做错 → 过几个知识点后再出现

## 二、UI设计

### 界面布局

#### 卡片正面
```
┌─────────────────────────────┐
│  ← 返回          3/15   ⚙️  │  顶部栏
├─────────────────────────────┤
│                             │
│    ┌───────────────────┐    │
│    │                   │    │
│    │    知识点名称       │    │  卡片正面
│    │    (点击翻转)       │    │
│    │                   │    │
│    └───────────────────┘    │
│                             │
│         ↑ ↓ ← →            │  手势提示
├─────────────────────────────┤
│  [不会]  [有点印象]  [会]   │  底部按钮（翻转前禁用）
└─────────────────────────────┘
```

#### 卡片背面
```
┌─────────────────────────────┐
│         (已翻转)              │
│    ┌───────────────────┐    │
│    │   知识解析文字      │    │
│    │                   │    │
│    │   📝 例题          │    │
│    │   (如果有关联题)    │    │
│    │                   │    │
│    │   💡 记忆技巧      │    │
│    │   (如果存在)       │    │
│    └───────────────────┘    │
│                             │
├─────────────────────────────┤
│  [不会]  [有点印象]  [会]   │  ← 表态按钮
│         或                  │
│    [↑上一题]  [↓下一题]     │  ← 表态后才能翻页
└─────────────────────────────┘
```

### 配色方案（简约现代风格）

| 元素 | 颜色 |
|------|------|
| 背景 | theme.bg（浅灰/白色） |
| 卡片 | theme.bgCard（白色） |
| 不会 | #ef4444（红色） |
| 有点印象 | #f59e0b（橙色） |
| 会 | #10b981（绿色） |
| 上一题 | #3b82f6（蓝色） |
| 下一题 | #3b82f6（蓝色） |
| 滑动反馈-左 | 微红渐变 |
| 滑动反馈-右 | 微绿渐变 |
| 滑动反馈-上下 | 微蓝渐变 |

### 动效设计
- 卡片翻转：3D翻转动画，300ms
- 按钮点击：轻微缩放 (scale: 0.95)
- 滑动手势：卡片跟随手指，带弹性回弹
- 翻页：淡入淡出切换

## 三、交互设计

### 手势操作
| 手势 | 操作 |
|------|------|
| 点击卡片 | 翻转 |
| 左滑 | 有点印象（带红色渐变反馈） |
| 右滑 | 会（带绿色渐变反馈） |
| 上滑 | 上一题 |
| 下滑 | 下一题 |

### 按钮操作
| 状态 | 按钮 |
|------|------|
| 翻转前 | [不会] [有点印象] [会]（点击翻转+表态） |
| 表态后 | [↑上一题] [↓下一题]（翻页按钮） |

### 翻页规则
- 必须先表态（选择不会/有点印象/会）才能翻页
- 表态后，按钮从表态变成翻页
- 选择上下题后，当前卡片翻回正面，可重新翻转重新表态

## 四、评分算法

### 单个知识点分数计算

| 学习次数 | 分数 |
|---------|------|
| 第1次 | 100 |
| 第2次 | 80 |
| 第3次 | 60 |
| 第4次 | 40 |
| ... | 递减 |

### 关联题目分数计算

| 做对次数 | 分数 |
|---------|------|
| 第1次 | 100 |
| 第2次 | 80 |
| 第3次 | 60 |
| ... | 递减 |

### 综合分数计算

```
有题目：综合分 = 闪记分×50% + 题目分×50%
无题目：综合分 = 闪记分
```

### 熟练度评定

| 综合分 | 熟练度 |
|-------|--------|
| 80+ | 熟练 |
| 60-79 | 一般 |
| 40-59 | 生疏 |
| <40 | 不会 |

### 分数衰减

```
新综合分 = 旧综合分 × 0.5 + 本次分 × 0.5
```

## 五、数据结构

### 类型定义

```typescript
// 学习记录
interface StudyRecord {
  date: string;           // ISO日期
  type: 'flashcard';      // 类型
  score: number;          // 得分 100/80/60/40...
  knowledgePointId: string;
}

// 题目记录
interface QuizRecord {
  date: string;           // ISO日期
  questionId: string;     // 题目ID
  correct: boolean;        // 是否正确
  score: number;           // 得分 100/80/60...
  knowledgePointId: string;
}

// 熟练度等级
type ProficiencyLevel = 'none' | 'rusty' | 'normal' | 'master';

// 知识点
interface KnowledgePoint {
  id: string;
  name: string;
  explanation: string;
  memoryTip?: string;       // 可选的记忆技巧
  subjectId: string;
  chapterId: string;
  proficiency: ProficiencyLevel;
  studyRecords: StudyRecord[];
  quizRecords: QuizRecord[];
  currentScore: number;     // 当前综合分
}
```
